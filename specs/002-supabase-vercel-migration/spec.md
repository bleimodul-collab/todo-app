# Feature Specification: demodev Tasks — 클라우드 인증 및 데이터 동기화

**Feature Branch**: `002-supabase-vercel-migration`
**Created**: 2026-05-22
**Status**: Draft
**Input**: User description: "기존 localStorage 기반 todo-app을 Supabase + Vercel로 마이그레이션. Supabase Auth로 Google/GitHub 소셜 로그인 실제 구현, Supabase DB(PostgreSQL)로 할 일 데이터 영속화, Vercel로 배포. 4개 화면(로그인/메인/캘린더/통계) UI는 그대로 유지하고 백엔드만 교체."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 계정으로 로그인한다 (Priority: P1)

사용자는 Google 또는 GitHub 계정으로 실제 인증을 거쳐 앱에 로그인한다. 로그인 후에는
어떤 기기·브라우저에서도 자신의 할 일 데이터에 접근할 수 있다.

**Why this priority**: 인증이 없으면 클라우드 저장·다기기 접근이 불가능하다. 이번
마이그레이션의 핵심 전제 조건이다.

**Independent Test**: 로그인 화면에서 Google 또는 GitHub 버튼을 클릭하면 해당 서비스의
동의 화면이 열리고, 인증 성공 후 메인 화면으로 이동하며 해당 계정의 할 일 목록이 표시된다.

**Acceptance Scenarios**:

1. **Given** 앱에 접속했을 때, **When** 로그인 화면에서 "Google로 로그인" 버튼을 클릭하면,
   **Then** Google 계정 선택·동의 화면이 열리고 완료 후 메인 화면("/main")으로 리다이렉트된다.
2. **Given** 앱에 접속했을 때, **When** 로그인 화면에서 "GitHub로 로그인" 버튼을 클릭하면,
   **Then** GitHub 인증 화면이 열리고 완료 후 메인 화면("/main")으로 리다이렉트된다.
3. **Given** 로그인하지 않은 상태에서, **When** 메인·캘린더·통계 URL에 직접 접근하면,
   **Then** 로그인 화면으로 리다이렉트된다.
4. **Given** 로그인된 상태에서, **When** 앱을 닫고 다시 열면, **Then** 별도 로그인 없이
   자동으로 메인 화면이 표시된다(세션 유지).
5. **Given** 소셜 로그인 OAuth 흐름 중 사용자가 취소하면, **Then** 로그인 화면으로
   돌아오고 오류 메시지 없이 다시 시도할 수 있다.

---

### User Story 2 - 할 일 데이터가 클라우드에 저장된다 (Priority: P1)

로그인된 사용자의 모든 할 일 변경(생성·수정·완료·삭제·서브태스크·즐겨찾기)이
클라우드에 저장된다. 브라우저를 닫거나 기기를 바꿔도 데이터가 유지되며,
다른 사용자의 데이터와 완전히 격리된다.

**Why this priority**: 클라우드 저장이 이번 마이그레이션의 핵심 가치다.

**Independent Test**: 로그인 후 할 일을 추가하고 브라우저를 완전히 닫는다. 다시 앱을
열면 로그인 없이(세션 유지) 또는 다시 로그인한 뒤 추가한 할 일이 그대로 표시된다.

**Acceptance Scenarios**:

1. **Given** 로그인된 상태에서 새 할 일을 추가했을 때, **When** 브라우저를 닫고 다시
   앱을 열면, **Then** 추가한 할 일이 그대로 표시된다.
2. **Given** 로그인된 상태에서 할 일을 완료·수정·삭제했을 때, **When** 새로고침하면,
   **Then** 변경사항이 100% 유지된다.
3. **Given** 사용자 A와 사용자 B 계정이 각각 다른 할 일을 가지고 있을 때, **When**
   사용자 A로 로그인하면, **Then** 사용자 A의 할 일만 표시되고 사용자 B의 데이터는
   보이지 않는다.
4. **Given** 처음 로그인하는 사용자(클라우드에 데이터 없음)일 때, **When** 메인
   화면을 보면, **Then** 기존 001-todo-app과 동일한 업무 테마 샘플 할 일이 시드되어
   표시된다.
5. **Given** 네트워크가 끊긴 상태에서 할 일 변경을 시도하면, **Then** 오류 메시지가
   표시되고 변경이 적용되지 않는다.

---

### User Story 3 - 로그아웃한다 (Priority: P2)

사용자는 앱에서 명시적으로 로그아웃하여 세션을 종료할 수 있다. 로그아웃 후에는
로그인 화면으로 이동하고, 재로그인 전에는 데이터에 접근할 수 없다.

**Why this priority**: 보안에 필수적이나 인증·저장 기능(P1) 위에 얹히므로 P2.

**Independent Test**: 로그인된 상태에서 로그아웃하면 로그인 화면으로 이동하고,
메인 URL에 직접 접근해도 로그인 화면으로 리다이렉트된다.

**Acceptance Scenarios**:

1. **Given** 로그인된 상태에서, **When** 로그아웃 버튼을 클릭하면, **Then** 세션이
   종료되고 로그인 화면으로 이동한다.
2. **Given** 로그아웃한 상태에서, **When** 메인·캘린더·통계 URL에 직접 접근하면,
   **Then** 로그인 화면으로 리다이렉트된다.
3. **Given** 로그인 세션이 만료된 상태에서, **When** 보호된 화면에 접근하면,
   **Then** 자동으로 로그인 화면으로 이동한다.

---

### User Story 4 - 공개 URL로 앱에 접근한다 (Priority: P2)

앱이 공개 URL로 배포되어 로컬 서버(`localhost`) 없이 어디서든 접근 가능하다.

**Why this priority**: 클라우드 저장의 자연스러운 완성. 기능 자체보다는 운영 관련이라 P2.

**Independent Test**: 배포 URL을 새 브라우저에서 열면 로그인 화면이 표시되고,
로그인 후 4개 화면의 모든 기능이 로컬과 동일하게 동작한다.

**Acceptance Scenarios**:

1. **Given** 앱이 배포되었을 때, **When** 공개 URL을 브라우저에 입력하면,
   **Then** 로그인 화면이 표시된다.
2. **Given** 공개 URL에서 로그인했을 때, **When** 4개 화면을 탐색하면,
   **Then** 모든 기능이 로컬 환경과 동일하게 동작한다.

---

### User Story 5 - 다기기 데이터 접근 (Priority: P3)

같은 계정으로 다른 기기에서 로그인하면 최신 데이터를 볼 수 있다. 페이지 로드 또는
새로고침 시 서버에서 최신 데이터를 가져온다(실시간 자동 동기화는 범위 밖).

**Why this priority**: P1의 클라우드 저장으로 기기 간 데이터 공유 자체는 달성된다.
P3는 그 경험의 확인 수준이다.

**Independent Test**: 두 탭에서 같은 계정으로 로그인 후 탭 A에서 할 일을 추가하고,
탭 B에서 새로고침(또는 실시간으로) 확인한다.

**Acceptance Scenarios**:

1. **Given** 기기 A에서 로그인 후 할 일을 추가했을 때, **When** 기기 B에서 같은 계정으로
   로그인하면(또는 새로고침하면), **Then** 추가한 할 일이 기기 B에도 표시된다.

---

### Edge Cases

- OAuth 인증 흐름 중 취소하면 오류 없이 로그인 화면으로 돌아온다.
- 네트워크 없이 데이터 변경을 시도하면 오류 메시지가 표시된다.
- 두 기기에서 동시에 같은 할 일을 수정하면 마지막 저장이 우선된다.
- 로그인 세션 만료 시 자동으로 로그인 화면으로 이동한다.
- 로그아웃 후 뒤로 가기 버튼으로 이전 화면 복원을 시도하면 로그인 화면으로 리다이렉트된다.

## Requirements *(mandatory)*

### Functional Requirements

#### 인증

- **FR-001**: 사용자는 Google 계정으로 소셜 로그인할 수 있어야 한다.
- **FR-002**: 사용자는 GitHub 계정으로 소셜 로그인할 수 있어야 한다.
- **FR-003**: 로그인하지 않은 사용자가 메인·캘린더·통계 화면 URL에 직접 접근하면
  로그인 화면으로 리다이렉트되어야 한다.
- **FR-004**: 로그인 성공 후 세션이 유지되어 브라우저를 닫고 다시 열어도 재로그인
  없이 메인 화면이 표시되어야 한다.
- **FR-005**: 사용자는 앱 내 어디서든 로그아웃할 수 있어야 하며, 로그아웃 후 로그인
  화면으로 이동해야 한다.
- **FR-006**: 로그인 세션 만료 시 자동으로 로그인 화면으로 이동해야 한다.
- **FR-007**: OAuth 인증 흐름 중 취소하면 오류 없이 로그인 화면으로 돌아와야 한다.

#### 데이터 영속화

- **FR-008**: 각 사용자의 할 일 데이터는 해당 사용자만 읽고 쓸 수 있어야 한다.
- **FR-009**: 할 일의 모든 변경(생성·수정·완료·삭제·서브태스크 CRUD·즐겨찾기)이
  클라우드에 저장되어야 한다.
- **FR-010**: 사용자가 로그인하면 이전 세션의 모든 할 일 데이터를 즉시 볼 수 있어야 한다.
- **FR-011**: 처음 로그인하는 사용자(클라우드에 데이터 없음)는 업무 테마 샘플
  할 일로 시작해야 한다(기존 001-todo-app 동일 방식).
- **FR-012**: 네트워크 오류로 데이터 저장에 실패하면 사용자에게 오류 메시지를
  표시해야 한다.
- **FR-013**: 테마 선택(라이트/다크)도 클라우드에 저장되어 다기기에서 동일하게
  적용되어야 한다.

#### UI / 화면

- **FR-014**: 4개 화면(로그인·메인·캘린더·통계)의 레이아웃·디자인·인터랙션은
  기존과 동일하게 유지되어야 한다.
- **FR-015**: 현재 로그인한 사용자의 이름 또는 이메일이 사이드 내비게이션에 표시되어야 한다.
- **FR-016**: 로그아웃 버튼은 사이드 내비게이션 또는 사용자 프로필 영역에서
  접근 가능해야 한다.

#### 배포

- **FR-017**: 앱은 공개 URL로 접근 가능해야 하며 로컬 서버 없이도 동작해야 한다.
- **FR-018**: 배포 환경에서도 인증·데이터 저장·4개 화면 기능이 모두 동작해야 한다.

### Key Entities

- **User (사용자)**: 인증된 앱 사용자. 속성 — 고유 id, 이메일, 표시 이름,
  소셜 프로바이더(Google/GitHub), 가입일.
- **Task (할 일)**: 기존 엔티티에 소유자 참조(user_id) 추가. 사용자 간 데이터 격리.
- **Subtask (서브태스크)**: 기존 엔티티. 부모 Task를 통해 소유자 격리.
- **Session (세션)**: 로그인 상태를 나타내는 인증 세션. 만료 시 자동 로그아웃.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자는 Google 또는 GitHub 버튼 클릭 후 60초 이내에 메인 화면 로그인을
  완료할 수 있다.
- **SC-002**: 로그인한 사용자의 모든 데이터 변경이 브라우저를 닫고 다시 열어도 100% 유지된다.
- **SC-003**: 로그인하지 않은 상태로 보호된 URL에 접근 시 100% 로그인 화면으로
  리다이렉트된다.
- **SC-004**: 배포 URL에서 4개 화면의 모든 인터랙션이 로컬 환경과 동일하게 동작한다.
- **SC-005**: 기존 4개 화면의 시각 디자인이 마이그레이션 후에도 100% 유지된다.
- **SC-006**: 두 계정의 데이터가 서로 노출되지 않는다 — A 계정 로그인 시 B 계정
  데이터가 0건 표시된다.

## Assumptions

- 이메일/비밀번호 인증은 구현하지 않는다. Google과 GitHub OAuth만 지원한다.
- 카테고리는 기존 5종 고정값을 유지한다. 사용자별 커스텀 카테고리는 범위 밖이다.
- "오늘" 기준일(2026-05-15 고정)은 마이그레이션 후에도 유지된다.
- 오프라인 상태에서의 로컬 편집 후 자동 동기화(오프라인 퍼스트)는 범위 밖이다.
- 팀/조직 기능, 할 일 공유, 협업 기능은 범위 밖이다.
- 통계 화면 데이터는 현재와 동일하게 고정 합성값을 유지한다. 실시간 통계 계산은 범위 밖이다.
- 로그아웃 버튼 위치는 사이드 내비게이션 하단(현재 테마 토글 옆)으로 추가한다.
